このシートを 上から順に1つずつ 確認してください。 全部 ☑ になったら最後の 「本番スタートを有効にする」 ボタンを押せます。

困ったらまずシステム管理者に連絡。スクショを添えると早いです。

管理画面URL：https://ikuno-isc.github.io/clinic-booking/admin-new.html

A. 画面を開く

  • 管理画面を開いてログインできた
  • URL：上のリンク
  • メールアドレスとパスワードでログイン

B. テストデータが残っていないか

  • テストメニュー を消した
  • お店の設定 → 💉 メニュー・料金

  • 「11111」「22222」「テスト」などの名前が残っていたら 「🗑 削除」

  • [ ] テスト担当アーティスト を消した

  • お店の設定 → 👩‍⚕️ アーティスト

  • 「テスト担当A」「テスト担当B」などが残っていたら停止 or 削除

  • [ ] テスト予約 を消した

  • 予約管理 → 期間タブを 「今後すべて」
  • 「テスト」と名の付く予約があれば 「❌ キャンセル」

C. 正式内容が揃っているか

  • メニュー の名前・料金・所要時間が正式版になっている
  • お店の設定 → 💉 メニュー・料金 で全件目視

  • 違うものは 「✏️ 設定を変更」 → 直して 「💾 保存」

  • [ ] アーティスト の名前・写真・担当メニューが全員分そろっている

  • お店の設定 → 👩‍⚕️ アーティスト

  • 担当メニューが1件も紐付いていないアーティスト は予約ページに出てきません。必ず紐付け確認

  • [ ] シフト が本番用に設定されている

  • お店の設定 → 🗓 シフト → 全アーティスト分を順に確認
  • 基本シフト（曜日パターン）と来月の例外日（休み・臨時出勤）

D. 予約ページが開くか

  • 予約ページ を実際に開いて、メニュー・担当・日時が選べる
  • お店の設定 → ℹ️ システム情報 に予約ページのリンクがある
  • そのリンクを別タブで開く
  • メニュー → 担当 → 日時 → 「予約する」まで進めるか確認
  • 開けない場合はシステム管理者へ連絡

予約ページURL（参考）： - 久留米：https://ikuno-isc.github.io/clinic-booking/new.html?store=kurume - 小倉：https://ikuno-isc.github.io/clinic-booking/new.html?store=kokura

E. LINE通知が届くか

  • テスト予約を1件入れて、LINEに通知が届いた
  • D で開いた予約ページで 自分の名前で1件予約 を入れる
  • 自分のLINEにお客様用の自動返信 が届くか
  • スタッフのLINE（管理用）にも通知 が来るか
  • 届かない場合はシステム管理者へ連絡

F. Googleカレンダーに入るか

  • テスト予約が Googleカレンダーに自動で書き込まれた
  • Googleカレンダー：https://calendar.google.com/
  • E で入れたテスト予約の日時にイベントが入っているか
  • 入っていない場合はシステム管理者へ連絡（gcal-sync 設定の確認）

G. 予約まわりの操作が動くか

  • 要対応 → 完了 ができる
  • 顧客カルテ → 「要対応」 フィルタ → テスト予約のお客様
  • 下の 「✏️ メモ・区分・有人対応の更新」 で有人対応を 「対応済み」

  • 「この内容で更新する」 → バッジが消えるか

  • [ ] キャンセル操作 が動く

  • 予約管理 → テスト予約の 「❌ キャンセル」
  • Googleカレンダーからも自動で消えるか確認

H. スタッフがログインできるか

  • スタッフアカウントで実際にログインできた
  • お店の設定 → 👥 スタッフ で発行済みのスタッフを確認
  • 本人にURLとパスワードを伝えて、実機でログインしてもらう
  • ログインできない場合 → 「有効/無効」が「有効」 になっているか確認

I. AIが変な案内をしないか

  • LINEに テストメッセージ を送って、AIが正しく返事する
  • LINEの公式アカウントで例：「眉アートメイクについて教えて」
  • 料金・メニュー名・予約リンクが 正しい内容 で返ってくるか
  • おかしい場合 → 画面スクショを撮ってシステム管理者へ連絡（ナレッジ修正が必要）

J. 最終 OK 欄（本番スタート）

  • A 〜 I が全部 ☑ になっている
  • お店の設定 → 🧭 初期設定
  • ステップ9「本番スタート」の横に 「本番スタートを有効にする」 ボタンが出ている
  • 出ていない場合は 1〜8 のどこかが未完了 → 各ステップの「ここを設定する →」リンクから修正
  • 「本番スタートを有効にする」 を押した
  • 「本番開始：可能」表示に変わった

本番開始後

困ったときはシステム管理者へ連絡（スクショ・状況メモを添えて）。 本番開始後の最初の予約は要観察。LINE通知・Googleカレンダー・要対応の流れがいつも通り動くか、最初の数件をしっかりチェックしてください。
